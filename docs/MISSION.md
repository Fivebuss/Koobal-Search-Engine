# Mission — Koobal Search Engine

> The mission of Koobal Search Engine is to organize the user's parts list and make it quickly and universally accessible and useful.

## Products

| Product | Role |
|---------|------|
| **Full Koobal Search Engine** | Full product: predictive dropdown, suggestions, history, later categories/V2, etc. **Plus** the same core native/stock search improvements (Enter-to-search halt, tightened results, SearchStart guards, later global bars, …). |
| **Koobal Native Search** | Standalone of **some** core Koobal features — the **native/stock search improvements** only: halt-until-Enter, tightened results, search guards, later global bars. Branding **only** on the **main** (editor) search bar. **No** predictive dropdown, history, or other full-Koobal features. |

**Ongoing policy:** keep Native Search updated with **any relevant fixes** from full Koobal so the shared native-search core stays in parity.

## Implications

Every product surface exists to serve that mission: predictive search and apply make parts and filters findable fast; search history returns the user to recent intent; category/subassembly suggestions organize navigation across the editor’s own taxonomy; native-search improvements (halt, tighten, guards — including later global search bars) make stock search deliberate and consistent; V2 slide-expand / optional rebuild and Settings tune how the parts list is laid out and controlled — all in service of organization, speed, universal access, and usefulness.

## Out of mission

Koobal does **not** aim to replace the whole editor UI, own vessel construction tooling unrelated to finding/organizing parts, or ship unrelated flight/mission utilities. Features that do not make the parts list more organized, faster to reach, more universally accessible, or more useful are out of scope.
